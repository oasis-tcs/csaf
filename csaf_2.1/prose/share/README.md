# CSAF v2.1 Prose — Delivery Items

This `share` folder holds all versioned delivery items built from the sources in `edit/`.  Every file here is generated by `make release` and
placed under configuration control so that changes between editorial rounds are traceable with standard diff tools and with `nide diff`.

## Artifacts

| File                            | Description                                                                                            |
|:--------------------------------|:-------------------------------------------------------------------------------------------------------|
| `csaf-v2.1-draft.md`            | GFM+ formatted prose — the "Elephant"; primary human-readable source for review                        |
| `csaf-v2.1-draft.html`          | Standalone HTML; rendered by pandoc and post-processed by `nide inject`                                |
| `csaf-v2.1-draft.pdf`           | PDF compiled via typst                                                                                 |
| `csaf-v2.1-draft.typ`           | Typst source file produced by pandoc; the direct input to `typst compile`                              |
| `csaf-v2.1-draft.ir.json`       | nide internal representation (IR) assembled from the source sections; channel-neutral content snapshot |
| `csaf-v2.1-draft.pdf.sha256`    | SHA-256 integrity checksum of the PDF; always present                                                  |
| `csaf-v2.1-draft.pdf.blake3`    | BLAKE3 integrity checksum of the PDF; present when `b3sum` was available at build time                 |
| `csaf-v2.1-draft.manifest.json` | nide content manifest — per-channel hashes and XMP metadata for all delivery items                     |

### Checksum files

`csaf-v2.1-draft.pdf.sha256` is always written by `make pdf` using `shasum -a 256`.
`csaf-v2.1-draft.pdf.blake3` is written in parallel when `b3sum` is installed.  If the blake3 file is present but `b3sum` is absent on your machine, the build will warn:

    WARNING: b3sum not installed — ../share/csaf-v2.1-draft.pdf.blake3 is stale.

The social contract among editors is that everyone installs `b3sum` (`cargo install b3sum`) so both checksum files stay current.
The tools do not enforce this — the warning is the only signal.

To verify manually:

    shasum -a 256 --check csaf-v2.1-draft.pdf.sha256
    b3sum --check csaf-v2.1-draft.pdf.blake3   # requires b3sum

## Building

All build targets are run from inside `edit/`:

    make gfm-plus   # assemble + render GFM+ and IR
    make html       # render HTML (implies gfm-plus)
    make pdf        # render PDF + typst source + checksums
    make release    # html + pdf + nide manifest + nide validate

## Comparing versions

`make diff` and `make diff-repro` use `nide diff`, which extracts the
previous committed version from the Fossil repository automatically — no temporary files, no manual VCS spelunking.

    make diff        # prose diff of the HTML against VCS HEAD
    make diff-repro  # reproducible diff of the PDF against VCS HEAD

For finer control, call `nide diff` directly from `edit/`:

    # prose mode — readable word-level diff of the HTML
    nide diff --mode prose ../share/csaf-v2.1-draft.html

    # repro mode — suppress timestamp noise; useful for PDF round-trips
    nide diff --mode repro ../share/csaf-v2.1-draft.pdf

    # meta mode — show only XMP/header metadata changes
    nide diff --mode meta ../share/csaf-v2.1-draft.pdf

    # prose diff with XMP block appended
    nide diff --mode prose+xmp ../share/csaf-v2.1-draft.html

    # full mode — content + blobs + timestamps + XMP
    nide diff --mode full ../share/csaf-v2.1-draft.html

Exit codes follow the `nide diff` convention: `0` identical, `1` differs, `2` usage or tool error.
